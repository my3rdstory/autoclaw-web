# AGENTS.md (autoclaw-web)

이 문서는 **오토클로 코딩 에이전트**가 이 코드베이스에서 작업할 때 따라야 하는 규칙/워크플로우다.

## 기본 원칙
- 목표: 사용자가 요청한 기능/버그를 **직접 수정**하고, **한글 커밋 메시지로 자동 커밋**까지 하는 것을 기본으로 한다.
- 단, **자동 Push/배포는 절대 하지 않는다.** (Push/배포는 사용자가 명시적으로 요청할 때만)
- 외부 코딩 CLI(Claude Code/Gemini/Codex/OpenCode/Pi 등) 실행/제안 금지.

## 작업 범위(레포/환경)
- 로컬 레포 경로: `/Users/dev/Apps/autoclaw-web`
- 이 레포는 원격 git(ssh) 연동이 이미 되어 있다고 가정한다.

## 작업 절차(기본)
1) 현 상태 확인
   - `git status`, `git branch --show-current`, `git remote -v` 확인
2) 변경 만들기
   - 관련 코드/리소스 수정
   - 필요한 최소한의 로컬 확인(예: 린트/테스트/빌드)이 있다면 수행
3) 커밋
   - 커밋 메시지는 **한글**로, 한 줄 요약 + 필요하면 2~3줄 설명
   - 예: `인증 화면 bootstrapHint 영문 번역 누락 수정`
4) 결과 보고
   - 무엇을 고쳤는지(파일/핵심 diff)와 어떻게 검증했는지 간단히 보고

## 안전/가드레일
- 토큰/키/서버 주소 등 민감정보를 커밋에 포함하지 않는다.
- 대규모 리팩토링은 요청이 있을 때만.
- 의심스러운 변경(보안 설정, 인증 로직, 외부 접근 정책)은 자동으로 진행하지 말고 메인 세션에 리스크를 먼저 보고한다.

## 이슈 메모
- 언어 토글 관련: 현재 UI는 자체 I18N 테이블 + `applyI18n()` 방식으로 보이며, 일부 DOM 텍스트(예: `#bootstrapHint`)가 업데이트 누락될 수 있다. 이런 경우 I18N 키 추가 + `applyI18n` 갱신 대상으로 포함하는 방식으로 고친다.
